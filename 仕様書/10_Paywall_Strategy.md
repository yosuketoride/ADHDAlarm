# 10. Paywall戦略（本人向け / 家族向け）

## 目的

- PROの価値を、本人と家族それぞれの文脈でわかりやすく伝える
- ただし商品は1つに保ち、課金構造を複雑に見せない
- 「どちらが課金しても相手にもメリットがある」ことを明確にする
- 家族側ユーザーが「事前通知って自分に関係あるの？」と混乱しないようにする

## 結論

- 商品は `PRO` の1つだけにする
- Paywall画面は `本人向け` と `家族向け` の2つに分ける
- 2つのPaywallで変えるのは `コピー・並び順・最初に強く伝える価値` だけ
- 価格、購入処理、権利の考え方は共通にする

## 基本ルール

### 1. 商品は分けない

- `PRO(本人)` と `PRO(家族)` のような別商品に見せない
- App Storeの商品、価格、購入導線は共通の `PRO` に統一する

理由:

- 実際の権利モデルは「どちらか1人がPROなら連携機能が使える」であり、商品を分けると誤解を生む
- ユーザーが理解したいのは「自分が払うと何が良くなるか」であって、「どの種類のPROを買うか」ではない

### 2. Paywallは2画面に分ける

- `本人向けPaywall`
- `家族向けPaywall`

理由:

- 本人と家族では刺さる悩みも欲しい価値も違う
- 同じ機能一覧をそのまま見せると、対象外の機能がノイズになって理解しづらい

### 3. 両方のPaywallに必ず入れる一文

以下は必須の共通説明とする。

`本人か家族、どちらか1人がPROなら、連携機能が使えます。`

必要に応じて補足文も入れる。

- `あなたがPROでも、相手にも連携のメリットがあります`
- `相手がPROでも、あなたは連携機能を使えます`

## 権利モデルの見せ方

### アプリ内の実際の見せ方

Paywallでは以下の3分類で説明する。

1. `本人にうれしいこと`
2. `家族にうれしいこと`
3. `どちらか1人がPROなら使えること`

これにより、

- 商品は1つ
- 便益は役割ごとに整理

という状態を作る。

### 避けるべき見せ方

- `PRO(本人)` `PRO(家族)` のように商品が2つあるように見せる
- 本人向け画面で見守り機能ばかりを先頭に出す
- 家族向け画面で事前通知やカレンダーだけを先頭に出す
- ORルールを書かず、機能だけを列挙する

## 本人向けPaywall

### 役割

- 本人が「自分の毎日がラクになる」と感じられるようにする
- そのうえで、家族との連携も広がることを補足する

### 最初に強く伝える価値

本人向けPaywallの価値3つは以下で固定する。

1. `予定に、もっと早く気づける`
2. `いつものカレンダーからそのまま使える`
3. `家族にサポートしてもらいやすくなる`

理由:

- ADHDユーザーの一番の痛みは「気づいたら遅かった」
- 早期通知が最も刺さりやすい
- 3つ目に家族メリットを自然に含めることで ORルールへの橋渡しになる

### 見出し案

- `大事な予定に、確実に気づける`

### 共感コピー案

- `予定に気づくのが直前になりがち`
- `通院やお薬をうっかり忘れてしまう`
- `家族に頼りたいけど、毎回お願いするのは気が引ける`

### ヒーロー補足文

- `カレンダーの予定をそのままアラーム予定に`

### 価値3つに対応する機能

1. `予定に、もっと早く気づける`
   - 事前通知複数回
2. `いつものカレンダーからそのまま使える`
   - カレンダー連携
   - Appleカレンダーから取り込む
3. `家族にサポートしてもらいやすくなる`
   - 家族との連携機能
   - 家族から予定を送ってもらえる

### 相手側へのメリット説明

本文中に以下のような補足を入れる。

- `あなたがPROなら、家族は予定送信や見守りを使いやすくなります。`

### 比較表の行名

- `本人向け: 事前通知`
- `本人向け: カレンダー自由選択`
- `本人向け: Appleカレンダー取り込み`
- `本人向け: 家族の生声アラーム`
- `連携: 家族から予定を送ってもらう`
- `連携: 異変を家族へ知らせる`

## 家族向けPaywall

### 役割

- 家族が「離れていても支えられる」と感じられるようにする
- そのうえで、本人側にも良い変化があることを補足する

### 最初に強く伝える価値

家族向けPaywallの価値3つは以下で固定する。

1. `予定をワンタップで届けられる`
2. `5分反応がなければ、すぐ気づける`
3. `7日分の様子を、まとめて確認できる`

理由:

- 家族の不安は「何かあったとき気づけるか」
- 2番目の異変通知が最も感情に刺さる
- ただし最初に「自分から動ける」送る機能を置くことで能動感を出す
- 受動的な安心より、能動的に支えられる感覚を先に見せる

### 見出し案

- `離れていても、ご家族の予定を届けて見守れる`

### 共感コピー案

- `ちゃんとお薬を飲めているか心配`
- `通院の日を忘れていないか不安`
- `何かあったとき、すぐ気づけないのがこわい`

### ヒーロー補足文

- `通院やお薬の予定を送り、もしものときも気づきやすくなります`

### 価値3つに対応する機能

1. `予定をワンタップで届けられる`
   - 予定送信
2. `5分反応がなければ、すぐ気づける`
   - 異変通知
   - LINEでお知らせ
3. `7日分の様子を、まとめて確認できる`
   - 記録閲覧
   - Last Seen の確認

### 相手側へのメリット説明

本文中に以下のような補足を入れる。

- `あなたがPROなら、本人側でも事前通知やカレンダー機能が使いやすくなります。`

### 比較表の行名

- `家族向け: 予定を送る`
- `家族向け: LINEで異変をお知らせ`
- `家族向け: 7日間の記録を見る`
- `家族向け: 最終確認時刻を見る`
- `本人向け: 事前通知`
- `本人向け: カレンダー自由選択`

## 比較表の方針

### やること

- 比較表は `Paywall本体の必須要素` にはしない
- 列は基本的に `無料 / PRO` のままでよい
- 表示する場合は各行名に `本人向け` `家族向け` `連携` のラベルをつける

### ねらい

- 商品が1つであることを保つ
- その機能が誰に効くかを一目で伝える
- ただし、モバイルのPaywallで情報量が重くなりすぎるのを避ける

### 優先順位

- Paywall本体: 原則として比較表を出さない
- 詳細情報: `くわしく見る` などの折りたたみ領域に入れる
- 料金比較ページや設定内の説明画面: 必要なら表形式で出してよい

### 例

| 機能 | 無料 | PRO |
| --- | --- | --- |
| 本人向け: 事前通知 | - | ○ |
| 本人向け: カレンダー自由選択 | - | ○ |
| 本人向け: Appleカレンダー取り込み | - | ○ |
| 本人向け: 家族の生声アラーム | - | ○ |
| 家族向け: 予定を送る | - | ○ |
| 家族向け: LINEで異変をお知らせ | - | ○ |
| 家族向け: 7日間の記録を見る | - | ○ |
| 家族向け: 最終確認時刻を見る | - | ○ |
| 連携: どちらか1人がPROなら使える | - | ○ |

## UI構成の推奨順序

### 本人向け / 家族向け共通

モバイルのPaywall sheetでは、以下を基本形とする。

1. ヒーロー
2. 主役の価値を3つ
3. 連携条件の小バナー
4. `相手側にもこういうメリットがあります` の短い補足
5. 価格とCTA

### 連携条件バナーの扱い

- `本人か家族、どちらか1人がPROなら、連携機能が使えます`

これは仕組み説明であり、見出しの主張ではない。
主役コピーのすぐ横や直下で主張と同列に扱わず、価値3つの後に小さく補足する。

### 詳細情報の扱い

以下は初期表示ではなく、必要時のみ見せる。

- `本人にうれしいこと` の全項目
- `家族にうれしいこと` の全項目
- 比較表

表示方法は `アコーディオン` を標準とする。

理由:

- Paywall sheetは「閉じる」が常に近く、別画面遷移は離脱につながりやすい
- アコーディオンなら同じ文脈のまま詳細を見せられる
- CTAを視野に残したまま、必要な人だけ掘って読める

### なぜ削るか

- モバイルのPaywallは長くなるほど離脱しやすい
- 比較表は理解を助ける一方で、最初の意思決定には重すぎることが多い
- まずは `自分に関係ある価値` と `どちらか1人がPROでよい` を最短で伝える方が重要

## 実装方針

### 1. Paywallは2コンポーネントに分ける

- `PaywallView` は本人向け
- `FamilyPaywallView` は家族向け

### 2. 購入先は共通

- どちらから開いても最終的には同じ `PRO` を購入する
- StoreKitの商品、価格、購入処理、権利反映は共通

### 3. どちらにも相手側メリットを明示する

必須で入れる。

- 家族向けPaywall:
  - `あなたがPROなら、本人側でも便利機能が使いやすくなります`
- 本人向けPaywall:
  - `あなたがPROなら、家族も予定送信や見守りのメリットを受けられます`

### 4. 家族向けWelcomeとPaywallの役割を分ける

- `FamilyWelcomeView`:
  - オンボーディング
  - 家族としての価値を理解してもらう
  - 無料でペアリングするか、PROを見るかを選ぶ
- `FamilyPaywallView`:
  - 家族向けに最適化した課金説明
  - ただし最終購入対象は共通の `PRO`

### 5. 家族向けWelcomeから本人向けPaywallへ直接飛ばさない

原則として避ける。

理由:

- Welcomeの文脈は家族向けであり、その直後に本人向けの主張へ切り替わると違和感が強い
- ただし、共通Paywall戦略に統合し、文言をロール別セクションに整理した単一Paywallに将来統合するなら再検討可

### 6. 情報量を減らす

以下は削る、または弱める。

- 大きい共感カード
- 絵文字の箇条書き
- 大きい価値カード3枚
- 初期表示の比較表
- 色数の多いアイコン
- 長い補足文

### 7. 詰めるポイント

- ヒーロー高さを小さくする
- 見出し下の補足は1文までにする
- 価値3つはカードではなく、短い3行リストにする
- 1項目ごとの縦paddingを減らす
- CTAまでの距離を短くする

### 8. アイコン方針

- 絵文字は使わない、または最小限にする
- 基本は SF Symbols に統一する
- アイコン色はモノクロ寄りで目立たせすぎない
- 本当に強調したい要素だけ色を使う

### 9. 画像方針

- おばあちゃん写真は、本人向け / 家族向けのどちらにも使わない
- ふくろう画像、または抽象的なイラストを使う
- 画像は主役にしすぎず、コピーの補助にとどめる

## 今回の意思決定

現時点では以下を推奨する。

- Paywallは2つに分ける
- 商品は1つに保つ
- どちらの画面にも ORルールを明記する
- 機能は `本人向け / 家族向け / 連携` に分類して見せる
- Paywall本体は `共感 + 価値3つ + ORルール + 相手側メリット + CTA` に圧縮する
- 比較表は詳細情報として必要時のみ見せる

## 文言サンプル

### 共通の重要文言

- `本人か家族、どちらか1人がPROなら、連携機能が使えます。`

### 家族向け補足

- `あなたがPROでも、本人側に便利機能のメリットが届きます。`

### 本人向け補足

- `あなたがPROでも、家族は予定送信や見守りのメリットを受けられます。`

## メモ

- 家族側ユーザーが `事前通知って自分に関係あるの？` と感じるのは自然であり、UIの問題として解決すべき
- 解決策は「商品を分けること」ではなく、「便益を役割別に整理して見せること」
- もっとも避けるべき失敗は、1商品なのに2商品に見えること、または1商品すぎて誰に効く機能かわからないこと
