## アーキテクチャ概要

```
RootView
├── OnboardingContainerView（初回のみ）
│   ├── HookView（掴み画面）
│   ├── MagicDemoView（デモ体験）
│   ├── PermissionsCTAView（権限一括取得）
│   └── WidgetGuideView（ウィジェット設置ガイド）
│       └── PaywallView（オンボーディング完了後に1回表示）
└── DashboardView（メイン）
    ├── 次のアラームカード（カウントダウン）
    ├── 今日の予定リスト
    ├── 音声入力FABボタン → MicrophoneInputView
    └── 設定画面（歯車アイコン）
```

**技術スタック**
- **アラーム**: AlarmKit（iOS 26専用・マナーモード貫通。Apple標準の時計アラームと同等の信頼性）
- **カレンダー**: EventKit（iPhoneの標準カレンダーアプリ連携。Google CalendarやOutlookは不使用）
- **音声合成**: AVSpeechSynthesizer（オンデバイス・サーバー代ゼロ）
- **音声認識**: SFSpeechRecognizer（日本語・オンデバイス）
- **課金**: StoreKit 2
- **ウィジェット**: WidgetKit
- **SOS通知**: Supabase（Edge Functions + Realtime）
- **UI**: SwiftUI + @Observable / async-await

## ✅ 実装済み機能一覧

### 1. オンボーディング（初回起動チュートリアル）

4ステップのフルスクリーンチュートリアル。完了後にPaywallを1回表示。

| ステップ | 画面 | 内容 |
|---------|------|------|
| 1 | **HookView** | インパクトのあるキャッチコピーでアプリの価値を一瞬で伝える |
| 2 | **MagicDemoView** | 実際にデモアラームを体験できる（「あとで試す」スキップ可） |
| 3 | **PermissionsCTAView** | カレンダー・AlarmKit・マイク・音声認識の権限を一括取得 |
| 4 | **WidgetGuideView** | PhaseAnimatorで4ステップのウィジェット設置手順をアニメーション |

- スワイプで進む形式（最後のページのみ「はじめる！」ボタン表示）
- 既にPRO購入済みの場合はPaywallをスキップ

### 3. 予定登録（Write-Throughアーキテクチャ）

**データフロー**: アプリ → EventKit（iPhoneカレンダー） → AlarmKit の一方向のみ。外部カレンダーからの吸い込みは禁止。

| 機能 | 詳細 |
|------|------|
| iPhoneカレンダーへの書き込み | EventKitを使用。イベントのメモ欄にUUIDマーカー(`<!-- wasure-bou:{UUID} -->`)を埋め込み、アプリが作成したイベントのみを管理 |
| AlarmKit登録 | マナーモード貫通アラームをOSレベルで登録 |
| 音声ファイル生成 | AVSpeechSynthesizerでCAF形式の音声ファイルを生成し、アラーム音として使用 |
| 複数事前アラーム（PRO） | 30分前・15分前・5分前など複数タイミングを同時登録可能 |
| 登録失敗時ロールバック | EventKit/AlarmKit/音声ファイルをすべて自動削除 |
| 確認カード表示 | 登録前に「解析結果の確認」画面を表示、日時・タイトルを編集可能 |

### 4. アラーム鳴動（RingingView）

| 機能 | 詳細 |
|------|------|
| AlarmKit発火検知 | AsyncSequenceでバックグラウンド/フォアグラウンド問わず常時監視 |
| フルスクリーン表示 | アプリ起動中はフルスクリーンのRingingViewを表示 |
| カスタム音声再生 | 生成済みCAFファイルをAVAudioPlayerで再生（6秒ごとループ） |
| スピーカー強制出力 | マナーモード中でもスピーカーから音声が出る |
| 音声フォールバック | CAFファイルがない場合はAVSpeechSynthesizerでリアルタイム読み上げ |
| 停止ボタン（1ボタン） | スヌーズなし。大きな緑のボタン1つ。 |
| 停止後メッセージ | 「そろそろ出発しましょう！」などポジティブなフィードバック |
| スワイプ離脱禁止 | `.interactiveDismissDisabled(true)` |
| SOS自動発信（PRO） | 5分間アラームを止めない場合、登録済みの家族連絡先にSOS通知 |

### 5. SOS・見守り機能（PRO）

Supabaseをバックエンドとして使用。カメラ監視なしの「非監視型見守り」。

| 機能 | 詳細 |
|------|------|
| ペアリングコード生成 | 4桁のコードを10分間有効で生成。家族が同じコードを入力して連携 |
| リアルタイム連携確認 | Supabase Realtimeで即時通知 + 3秒ポーリングのフォールバック |
| SOS送信 | アラームを5分以上止めない場合、Supabase Edge Function経由でSOS通知 |
| 家族への通知内容 | 「（イベント名）のアラームが○分間止まっていません」 |
| 設定画面 | 見守り連絡先の登録・ペアリング管理 |

### 13. UX・アクセシビリティ配慮

- タップターゲット最低60pt（高齢者対応）
- スワイプ操作にはタップ代替ボタンを用意
- ポジティブなエラーメッセージ（「遅刻です！」→「少し過ぎてしまいましたが、慌てずに向かいましょう！」）
- カウントダウン：残り10分以上は秒を非表示（ADHD向け不安軽減）
- 削除操作は必ず確認ダイアログ＋Undo対応
- IT用語を使わない（「同期」→「読み込む」「デフォルト」→「いつもの設定」）
